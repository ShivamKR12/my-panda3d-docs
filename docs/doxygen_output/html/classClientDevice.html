Detailed Description

Any of a number of different devices that might be attached to a ClientBase, including trackers, etc. This is an abstract interface; the actual implementations are in ClientTrackerDevice, etc.

Constructor & Destructor Documentation

◆ ~ClientDevice()

ClientDevice::~ClientDevice ( )
virtual

We don't actually call disconnect() at the ClientDevice level destructor, because by the time we get here we're already partly destructed. Instead, we should call disconnect() from each specific kind of derived class.

Member Function Documentation

◆ disconnect()

void ClientDevice::disconnect ( )

Disconnects the ClientDevice from its ClientBase object. The device will stop receiving updates.

Normally, you should not need to call this explicitly (and it is probably a mistake to do so); it will automatically be called when the ClientDevice object destructs.

The lock should not be held while this call is made; it will explicitly grab the lock itself.

◆ do_poll()

void ClientDevice::do_poll ( )
finalvirtual

Causes the connected ClientBase to poll all of its clients, if necessary. This will be a no-op if the client is running in forked mode, or if it has already polled everything this frame.

This should generally be called before accessing the data in this ClientDevice to ensure that it is fresh.
