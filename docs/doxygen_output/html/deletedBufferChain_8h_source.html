26// Though it's tempting, it doesn't seem to be possible to implement
27// DeletedBufferChain via the atomic exchange operation. Specifically, a
28// pointer may be removed from the head of the chain, then the same pointer
29// reinserted in the chain, while another thread is waiting; and that thread
30// will not detect the change. So instead, we always use a mutex.

32#ifndef NDEBUG
33// In development mode, we define USE_DELETEDCHAINFLAG, which triggers the
34// piggyback of an additional word of data on every allocated block, so we can
35// ensure that an object is not double-deleted and that the deleted chain
36// remains intact.

38#endif // NDEBUG
