144 // An expiration queue is a list of SoundData that are no longer being used.
145 // They are kept around for a little while, since it is common to stop using
146 // a sound for a brief moment and then quickly resume.

152/*
153 * An AudioSound that uses a SoundData is called a "client" of the SoundData.
154 * The SoundData keeps track of how many clients are using it. When the
155 * number of clients drops to zero, the SoundData is no longer in use. The
156 * expiration queue is a list of all SoundData that aren't in use, in least-
157 * recently-used order. If a SoundData in the expiration queue gains a new
158 * client, it is removed from the expiration queue. When the number of sounds
159 * in the expiration queue exceeds the cache limit, the first sound in the
160 * expiration queue is purged.
161 */
