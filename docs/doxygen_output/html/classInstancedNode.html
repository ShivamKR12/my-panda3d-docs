Detailed Description

This is a special node that instances its contents using a list of transforms that get applied on top of the node's own transform. This is a bit more limited than the regular instance_to mechanism, but it is a better choice for hardware instancing.

For best performance, it is highly recommended to flatten the nodes under this (by calling flatten_strong()), since culling will not be performed for individual sub-nodes under each instance.

Since
1.11.0

◆ combine_with()

PandaNode * InstancedNode::combine_with ( PandaNode *  other)
overridevirtual

Collapses this node with the other node, if possible, and returns a pointer to the combined node, or NULL if the two nodes cannot safely be combined.

The return value may be this, other, or a new node altogether.

This function is called from GraphReducer::flatten(), and need not deal with children; its job is just to decide whether to collapse the two nodes and what the collapsed node should look like.
