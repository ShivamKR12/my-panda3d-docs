\doxysubsection{Detailed Description}
Any of a number of different devices that might be attached to a \doxylink{classClientBase}{Client\+Base}, including trackers, etc. This is an abstract interface; the actual implementations are in \doxylink{classClientTrackerDevice}{Client\+Tracker\+Device}, etc.

Disconnects the \doxylink{classClientDevice}{Client\+Device} from its \doxylink{classClientBase}{Client\+Base} object. The device will stop receiving updates.

Normally, you should not need to call this explicitly (and it is probably a mistake to do so); it will automatically be called when the \doxylink{classClientDevice}{Client\+Device} object destructs.

{\footnotesize\ttfamily void Client\+Device\+::do\+\_\+poll (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [final]}, {\ttfamily [virtual]}}

Causes the connected \doxylink{classClientBase}{Client\+Base} to poll all of its clients, if necessary. This will be a no-\/op if the client is running in forked mode, or if it has already polled everything this frame.

This should generally be called before accessing the data in this \doxylink{classClientDevice}{Client\+Device} to ensure that it is fresh.
