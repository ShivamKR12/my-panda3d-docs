\doxysubsection{Detailed Description}
\doxylink{classNativeNumericData}{Native\+Numeric\+Data} and \doxylink{classReversedNumericData}{Reversed\+Numeric\+Data} work together to provide a sneaky interface for automatically byte-\/swapping numbers, when necessary, to transparency support big-\/endian and little-\/endian architectures.

Both of these classes provide interfaces that accept a pointer to a numeric variable and the size of the number, and they can append that data to the end of a string, or memcpy it into another location.

The difference is that \doxylink{classNativeNumericData}{Native\+Numeric\+Data} simply passes everything through unchanged, while \doxylink{classReversedNumericData}{Reversed\+Numeric\+Data} always byte-\/swaps everything. Otherwise, they have the same interface.

The transparent part comes from Little\+Endian and Big\+Endian, which are typedeffed to be one of these or the other, according to the machine\textquotesingle{}s architecture.
