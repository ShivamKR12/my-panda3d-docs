\doxysubsection{Detailed Description}
A View boils down the frame data to a linear list of times spent in a number of different Collectors, within a particular thread. This automatically accounts for overlapping start/stop times and nested Collectors in a sensible way.

{\footnotesize\ttfamily void PStat\+View\+::constrain (\begin{DoxyParamCaption}\item[{int}]{collector,  }\item[{bool}]{show\+\_\+level }\end{DoxyParamCaption})}

Changes the focus of the View. By default, the View reports the entire time for the frame, and all of the Collectors that are directly parented to "{}\+Frame"{}. By constraining the view to a particular collector, you cause the View to zoom in on that collector\textquotesingle{}s data, reporting only the collector and its immediate parents.

When you constrain the view, you may also specify whether the view should show time data or level data for the indicated collector. If level data, it reports the levels for the collector, and all of its children; otherwise, it collects the elapsed time.

Changing the constraint causes the current frame\textquotesingle{}s data to become invalidated; you must then call \doxylink{classPStatView_a58673e539f32b9df52b690e82a8fddbb}{set\+\_\+to\+\_\+frame()} again to get any useful data out. \Hypertarget{classPStatView_af3deba6eb7bb9997142ee6f353fdeec8}\label{classPStatView_af3deba6eb7bb9997142ee6f353fdeec8}
