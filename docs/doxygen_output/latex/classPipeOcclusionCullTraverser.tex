\doxysubsection{Detailed Description}
This specialization of \doxylink{classCullTraverser}{Cull\+Traverser} uses the graphics pipe itself to perform occlusion culling. As such, it\textquotesingle{}s likely to be inefficient (since it interferes with the pipe\textquotesingle{}s normal mode of rendering), and is mainly useful to test other, CPU-\/based occlusion algorithms.

This cannot be used in a multithreaded pipeline environment where cull and draw are operating simultaneously.

It can\textquotesingle{}t be defined in the cull subdirectory, because it needs access to \doxylink{classGraphicsPipe}{Graphics\+Pipe} and \doxylink{classDisplayRegion}{Display\+Region} and other classes in display. So we put it in grutil instead, for lack of any better ideas.

This callback function is intended to be overridden by a derived class. This is called as each \doxylink{classGeom}{Geom} is discovered by the \doxylink{classCullTraverser}{Cull\+Traverser}.

We do a sneaky trick in making \doxylink{classPipeOcclusionCullTraverser}{Pipe\+Occlusion\+Cull\+Traverser} inherit from both \doxylink{classCullTraverser}{Cull\+Traverser} and \doxylink{classCullHandler}{Cull\+Handler}--the traverser is its own handler! This is the normal callback into the traverser for rendering objects. We respond to this by firing off an occlusion test, and queuing up the object until the end of the scene.
