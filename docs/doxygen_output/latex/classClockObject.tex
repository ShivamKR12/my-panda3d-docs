\doxysubsection{Detailed Description}
A \doxylink{classClockObject}{Clock\+Object} keeps track of elapsed real time and discrete time. In normal mode, get\+\_\+frame\+\_\+time() returns the time as of the last time tick() was called. This is the "{}discrete"{} time, and is usually used to get the time as of, for instance, the beginning of the current frame.

In other modes, as set by set\+\_\+mode() or the clock-\/mode config variable, get\+\_\+frame\+\_\+time() may return other values to simulate different timing effects, for instance to perform non-\/real-\/time animation. See set\+\_\+mode().

In all modes, get\+\_\+real\+\_\+time() always returns the elapsed real time in seconds since the \doxylink{classClockObject}{Clock\+Object} was constructed, or since it was last reset.

You can create your own \doxylink{classClockObject}{Clock\+Object} whenever you want to have your own local timer. There is also a default, global \doxylink{classClockObject}{Clock\+Object} intended to represent global time for the application; this is normally set up to tick every frame so that its get\+\_\+frame\+\_\+time() will return the time for the current frame.
