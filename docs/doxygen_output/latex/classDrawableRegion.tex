\doxysubsection{Detailed Description}
This is a base class for \doxylink{classGraphicsWindow}{Graphics\+Window} (actually, \doxylink{classGraphicsOutput}{Graphics\+Output}) and \doxylink{classDisplayRegion}{Display\+Region}, both of which are conceptually rectangular regions into which drawing commands may be issued. Sometimes you want to deal with a single display region, and sometimes you want to deal with the whole window at once, particularly for issuing clear commands and capturing screenshots.

{\footnotesize\ttfamily void Drawable\+Region\+::set\+\_\+pixel\+\_\+zoom (\begin{DoxyParamCaption}\item[{PN\+\_\+stdfloat}]{pixel\+\_\+zoom }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

Sets the amount by which the pixels of the region are scaled internally when filling the image interally. Setting this number larger makes the pixels blockier, but may make the rendering faster, particularly for software renderers. Setting this number to 2.\+0 reduces the number of pixels that have to be filled by the renderer by a factor of 2.\+0. It doesn\textquotesingle{}t make sense to set this lower than 1.\+0.

It is possible to set this on either individual Display\+Regions or on overall Graphics\+Windows, but you will get better performance for setting it on the window rather than its individual Display\+Regions. Also, you may not set it on a \doxylink{classDisplayRegion}{Display\+Region} that doesn\textquotesingle{}t have both clear\+\_\+color() and clear\+\_\+depth() enabled.

This property is only supported on renderers for which it is particularly useful--currently, this is the tinydisplay software renderer. Other kinds of renderers allow you to set this property, but ignore it.
