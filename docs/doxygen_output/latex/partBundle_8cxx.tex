{\footnotesize\ttfamily PT (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{classAnimControl}{Anim\+Control}}}]{ }\end{DoxyParamCaption})}

Binds the animation to the bundle, if possible, and returns a new \doxylink{classAnimControl}{Anim\+Control} that can be used to start and stop the animation. If the anim hierarchy does not match the part hierarchy, returns NULL.

If hierarchy\+\_\+match\+\_\+flags is 0, only an exact match is accepted; otherwise, it may contain a union of Part\+Group\+::\+Hierarchy\+Match\+Flags values indicating conditions that will be tolerated (but warnings will still be issued).

If subset is specified, it restricts the binding only to the named subtree of joints.

The \doxylink{classAnimControl}{Anim\+Control} is not stored within the \doxylink{classPartBundle}{Part\+Bundle}; it is the user\textquotesingle{}s responsibility to maintain the pointer. The animation will automatically unbind itself when the \doxylink{classAnimControl}{Anim\+Control} destructs (i.\+e. its reference count goes to zero).

Binds an animation to the bundle. The animation is loaded from the disk via the indicated \doxylink{classLoader}{Loader} object. In other respects, this behaves similarly to bind\+\_\+anim(), with the addition of asynchronous support.

If allow\+\_\+aysnc is true, the load will be asynchronous if possible. This requires that the animation basename can be found in the \doxylink{classPartBundle}{Part\+Bundle}\textquotesingle{}s preload table (see get\+\_\+anim\+\_\+preload()).

In an asynchronous load, the animation file will be loaded and bound in a sub-\/thread. This means that the animation will not necessarily be available at the time this method returns. You may still use the returned \doxylink{classAnimControl}{Anim\+Control} immediately, though, but no visible effect will occur until the animation eventually becomes available.

You can test Anim\+Control\+::is\+\_\+pending() to see if the animation has been loaded yet, or wait for it to finish with \doxylink{classAnimControl_a71712dadc65e844ec234c0819fc518a8}{Anim\+Control\+::wait\+\_\+pending()} or even \doxylink{classPartBundle_a6826e9c58f49a810f8532cd397f6ba37}{Part\+Bundle\+::wait\+\_\+pending()}. You can also set an event to be triggered when the animation finishes loading with \doxylink{classAnimControl_a8dd637434968e3d9b045581dc2676d66}{Anim\+Control\+::set\+\_\+pending\+\_\+done\+\_\+event()}. \Hypertarget{partBundle_8cxx_a1fec9fe303e4044e8ba5a35d524119c6}\label{partBundle_8cxx_a1fec9fe303e4044e8ba5a35d524119c6}
